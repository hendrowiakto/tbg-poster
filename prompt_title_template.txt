<isi_rules>
"isi" is the descriptive body content for each variant. Build it as follows:

1. EXTRACT visual facts from the screenshots: characters/units owned with
   their rarity + variant/version labels, levels, weapons, currency amounts,
   account rank/level, achievements, content cleared, cosmetics. Cross-
   reference across multiple screenshots when info appears in more than one.

2. STANDOUTS: rank facts by rarity / value / hard-to-obtain. Lead "isi" with
   the strongest standout for that variant (limited/event-exclusive units,
   max-level rare units, top-tier currency stockpile, completed endgame, rare
   cosmetics).

   FOCUS_GUIDANCE OVERRIDE: If FOCUS_GUIDANCE in <context> contains user
   instructions (anything other than "(none ...)"), it OVERRIDES the default
   standout ranking. Apply FOCUS_GUIDANCE as the primary instruction for what
   to emphasize in "isi" across ALL variants. Example: if FOCUS_GUIDANCE says
   "Focus on rare characters and their constellations", every variant's "isi"
   should lead with character + constellation info, regardless of what else
   exists in the screenshots. If FOCUS_GUIDANCE conflicts with default
   standout logic, FOCUS_GUIDANCE wins.

3. DENSITY: pack "isi" with real facts. Length MUST be between ISI_MIN_CHARS
   and ISI_MAX_CHARS characters. If short, ADD more high-value facts. If long,
   DROP the least important facts. Never use filler words to pad length — only
   real visual facts.

4. VARIANT DIFFERENTIATION: each variant's "isi" must LEAD with a DIFFERENT
   standout. No two variants may paraphrase each other.
   Suggested rotation:
     Variant 1: rarest asset (named limited/event unit)
     Variant 2: breadth / collection completeness
     Variant 3: endgame / rank / clears
     Variant 4: resource / currency stockpile
     Variant 5+: cosmetic / hybrid angle
</isi_rules>

<hard_rules>
1. LANGUAGE: English only inside "isi".
2. VISUAL TRUTH ONLY: never invent facts. If unsure, omit.
3. UNIT COUNTING: count VISUALLY. Do not infer from achievement labels.
4. METADATA PRECISION: use exact version/variant names (e.g. "Summer Edition",
   "Anniversary").
5. NO MARKETING FLUFF: no "Amazing", "Best", "Godly", "Insane", "Cheap",
   "Must Buy", "Rare", "OP" — let facts speak.
6. Do NOT use GAME_NAME, its abbreviations, or partial names inside "isi".
7. "isi" must contain ONLY descriptive content — characters, levels, items,
   currencies, achievements, etc. It must NOT contain:
   - Any reference to the listing code or account ID.
   - Repeated copies of lookup values you already output in the "lookup_*"
     fields (e.g. don't put server name inside "isi" — it's already in
     "lookup_server").
   Inside "isi" you MAY (and SHOULD, if references use them) use:
     pipes "|"      as inner section separator
     commas ","     as item separator within a section
     slashes "/"    for "X/Y" pairs (e.g. "Hydro/Pyro DPS")
     parens "()"    around feature labels (e.g. "Hu Tao (C2)")
     plus "+"       for combinations (e.g. "Linnea+Sign", "9Limited+7")
   Match the punctuation style of REFERENCE_TITLES verbatim.

8. STRICT STYLE MIRRORING (highest priority for "isi"):
   "isi" MUST mimic the structure, separator pattern, and verbal density of
   REFERENCE_TITLES literally. If references use inner pipes "|" between
   sections, you MUST use the same. If references use abbreviations like
   "AR 58", "L90", "FR54", "McMale", "9Limited+7(*5Wep)", you MUST use those
   EXACT patterns — never write the full forms.

   FORBIDDEN FILLER WORDS / phrases (these never appear in references — they
   add nothing, just bloat length):
     "featuring", "with", "includes", "including", "offers", "holds", "such",
     "such as", "like", "for", "has", "this account", "an account", "account
     featuring", "and includes", "plus 10", "a roster of", "equipped with",
     "currently", "comes with", "Adventure Rank" (write "AR")

   FORBIDDEN PROSE STRUCTURE:
     - Full English sentences. Each section is a TERSE LABEL, not a sentence.
     - "Lv. 50", "Level 50", "Lvl 50" → write "L50" or "Lv50" (mirror what
       references use most).
     - "5-star weapons" → "*5Wep" if references abbreviate that way.
     - Numbered counts like "10 five-star weapons" → "10(*5Wep)" pattern.

   DENSITY TARGET: every character in "isi" must carry information. If you
   have spare characters within budget, ADD MORE FACTS, never pad with prose.

9. STYLE INFERENCE FROM REFERENCE_TITLES: before writing "isi", silently
   identify in REFERENCE_TITLES:
     - the inner separator (usually "|")
     - abbreviation conventions for level / rank / resources
     - section ordering (server → AR → team → counts → characters → resources
       → MC gender → trust signal)
     - punctuation density (parens around character groups, commas inside)
   Mirror these patterns. Your "isi" should be readable side-by-side with a
   reference title and look like the same author wrote it.
</hard_rules>
